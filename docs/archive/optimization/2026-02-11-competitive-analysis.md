# 2026-02-11 记忆系统竞争分析

## 竞品调研

今天调研了两个主要的个人AI记忆系统方案：

### 方案1：三层Cron系统（OpenClaw社区）

**核心架构**：
- Layer 1: Daily Context Sync（每晚11点自动捕获）
- Layer 2: Weekly Memory Compound（每周日晚上10点知识蒸馏）
- Layer 3: Hourly Micro-Sync（白天每3小时安全网）

**技术栈**：
- OpenClaw cron + QMD + Claude

**优势**：
- ✅ 轻量、自动化
- ✅ Hourly Micro-Sync避免重要决策延迟捕获

**劣势**：
- ❌ 无智能筛选（全量捕获，噪音越来越多）
- ❌ 无自动衰减（需要手动清理）
- ❌ 无元认知能力（无法区分确定/不确定）

---

### 方案2：PAI (Personal AI Infrastructure)

**核心架构**：
- TELOS系统：10个维度构建数字身份档案
- 三层记忆：Working Memory、Episodic Memory、Semantic Memory
- Hooks系统：让AI主动感知环境
- Skills系统：可组合的AI能力

**技术栈**：
- Markdown + 向量库 + Hooks + Skills

**优势**：
- ✅ 高度工程化、可编程
- ✅ TELOS身份档案让冷启动更快
- ✅ Hooks系统主动感知环境

**劣势**：
- ❌ 手动管理重
- ❌ 冷启动难（需要填写10个文件）
- ❌ 无智能筛选和衰减

---

## 我们的独特优势

对比两个方案，我们的记忆系统有以下独特优势：

### 1. 真正的"元认知能力"
```python
Fact: "用户对花生过敏，吃了会死"  # 100%确定
Belief: "我怀疑他在暗中阻挠项目"  # 60%置信度
```
AI可以区分"我记得的"和"我猜的"，避免把不确定信息当成事实。

### 2. 智能筛选 + 自动衰减
```python
identity_health_safety: importance=1.0  # 永久保存
temporary_emotion: importance=0.2     # 自然衰减
```
自动筛选重要信息，自动淡化琐碎内容。

### 3. 7阶段自动Consolidation
完整的认知流水线：提取→分类→去重→冲突检测→衰减→快照

### 4. 访问追踪 + 时间敏感
常用记忆权重更高，自动清理过期信息。

## 需要学习的地方

### 从方案1学习：Hourly Micro-Sync
**问题**：我们的Consolidation每天只跑一次（凌晨3点），重要决策延迟到晚上才捕获。

**改进**：添加白天5个时间点轻量检查（10点、13点、16点、19点、22点），只捕获真正重要的内容。

### 从方案2学习：TELOS身份档案
**问题**：我们有MEMORY.md，但结构化不够，AI冷启动慢。

**改进**：创建TELOS-lite，简化为5个核心维度：
- 00-identity.md（核心身份）
- 01-values.md（价值观）
- 02-goals.md（目标）
- 03-skills.md（技能）
- 04-communication.md（沟通风格）

### 从方案2学习：Hooks系统
**问题**：记忆系统是被动接收，不能主动感知环境。

**改进**：添加Hooks，主动捕获Git状态、TODO列表、日历等上下文。

## 差异化定位

**我们的定位**："会思考的记忆" vs "会存储的笔记"

| 维度 | 对比 |
|------|------|
| **三层Cron** | 自动化日志系统 |
| **PAI** | 个人知识管理 |
| **我们** | 智能认知系统 |

核心差异：
- 他们：记得多（全量存储）
- 我们：记得准（智能筛选）+ 记得久（自动衰减）+ 记得有用（元认知）
